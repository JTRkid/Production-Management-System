## 1. 项目背景与目标

### 1.1 背景

中小型制造企业在生产管理过程中面临以下痛点：

- 订单、计划、生产、质检、库存等环节信息孤岛，依赖 Excel 和纸质单据流转
- 生产进度不透明，无法实时掌握工单状态和产能利用率
- 质量数据分散，难以追溯不良品根因
- 库存数据滞后，经常出现物料短缺或积压

### 1.2 目标

构建一套覆盖全业务流程的生产管理系统，实现：

1. **订单驱动生产**：从客户订单到生产计划、工单派发、过程跟踪的全流程数字化
2. **质量全程管控**：检验标准统一管理，过程检验与完工检验数据可追溯
3. **库存实时可视**：出入库操作实时更新库存，支持多仓库管理
4. **数据辅助决策**：通过生产日报、完成率趋势、质量趋势等报表辅助管理决策

### 1.3 项目范围

| 范围 | 说明 |
|------|------|
| 包含 | 客户/供应商管理、销售/采购订单、生产计划、工单管理、报工跟踪、质量管理、库存管理、报表看板 |
| 不包含 | 财务核算、人力资源、设备维护工单、MRP 物料需求自动运算、排产甘特图 |

## 5. 系统接口概要

### 5.1 认证机制

- 登录接口：`POST /api/v1/auth/login/`，返回 JWT access_token + refresh_token
- 请求头：`Authorization: Bearer {access_token}`
- Token 刷新：`POST /api/v1/auth/token/refresh/`

### 5.2 API 统一规范

| 项目 | 规范 |
|------|------|
| 基础路径 | `/api/v1/{module}/` |
| 分页参数 | `?page=1&page_size=20` |
| 搜索参数 | `?search=关键词` |
| 筛选参数 | `?field=value`（如 `?status=draft`） |
| 排序参数 | `?ordering=-created_at` |
| 成功响应 | `{ "code": 200, "data": ... }` 或分页 `{ "count": N, "results": [...] }` |
| 错误响应 | `{ "code": 4xx, "message": "错误描述" }` |

### 5.3 模块接口清单

| 模块 | 路径前缀 | 主要资源 |
|------|----------|----------|
| 认证 | `/api/v1/auth/` | login, change-password, users, workshops, logs |
| 业务管理 | `/api/v1/business/` | customers, sales-orders, suppliers, purchase-orders |
| 基础数据 | `/api/v1/base-data/` | materials, bom-items, routes, route-steps, work-centers, equipment |
| 生产计划 | `/api/v1/prod-plan/` | production-plans |
| 工单管理 | `/api/v1/work-order/` | work-orders |
| 过程跟踪 | `/api/v1/prod-track/` | work-reports |
| 质量管理 | `/api/v1/quality/` | standards, records |
| 库存管理 | `/api/v1/inventory/` | warehouses, inventories, transactions |
| 报表看板 | `/api/v1/dashboard/` | overview, prod_daily, completion_rate, quality_trend |

## 6. 数据流图

### 6.1 核心业务数据流

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  客户档案  │───→│ 销售订单  │───→│ 生产计划  │───→│ 生产工单  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                      │
                    ┌───────────────────────────────────┤
                    ↓                                   ↓
              ┌──────────┐                       ┌──────────┐
              │ 采购订单  │                       │ 工序报工  │
              └──────────┘                       └──────────┘
                    │                                   │
                    ↓                                   ↓
              ┌──────────┐                       ┌──────────┐
              │ 入库操作  │                       │ 检验记录  │
              └──────────┘                       └──────────┘
                    │                                   │
                    ↓                                   ↓
              ┌──────────┐                       ┌──────────┐
              │ 库存更新  │←──────────────────────│ 出库操作  │
              └──────────┘                       └──────────┘
```

### 6.2 工单状态流转

```
待派工(pending) → 已派工(dispatched) → 生产中(in_progress) → 已完成(completed) → 已关闭(closed)
```

### 6.3 计划状态流转

```
草稿(draft) → 已审批(approved) → 已下发(released) → 已完成(completed)
                                                        ↗
                                              已取消(cancelled)
```

### 6.4 销售订单状态流转

```
草稿(draft) → 已审批(approved) → 生产中(in_production) → 已发货(shipped) → 已完成(completed)
                                                                              ↗
                                                                    已取消(cancelled)
```
